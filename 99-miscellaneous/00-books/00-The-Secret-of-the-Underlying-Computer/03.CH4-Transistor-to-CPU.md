## 4.2 What Does the CPU Do When It Is Idle?

### 4.2.1 What Is CPU Utilization?

### 4.2.2 Process Management and Scheduling
- The operating system assigns priorities to processes and places them into a queue so the scheduler can schedule them based on priority

```cpp
------------------------ ready queue
process   process   process                ----> CPU
------------------------                  |
^                                         |
|             ------------------------     |
I/O  <---- process    process        <-------
              ------------------------ blocking queue
```

### 4.2.3 Checking Queue State: A Better Design
- If the ready queue is empty, it means there are no processes for the OS to schedule and the CPU is in an idle state
- The queue can be kept non-empty so that the scheduler can always find a process to execute → System Idle Process
    - Always in the ready state
    - Has the lowest priority
    - When there are no schedulable processes, the scheduler runs this idle process

### 4.2.4 Everything Comes Back to the CPU
- There is a halt instruction designed for when the CPU enters an idle state
- This instruction reduces power consumption by putting some internal CPU modules into a low-power state
- It can only be executed in kernel mode
- Executing the halt instruction means there are no processes in the system that are ready to run

### 4.2.5 Idle Process and CPU Low-Power States
- The halt instruction is executed within the idle process (a loop continuously executing halt)
- If there are no schedulable processes, the scheduler runs the idle process
- The actual behavior of halt is more complex
    - It may enter deep sleep (C3) or deeper sleep (C4) states
    - The kernel must predict system idle time and decide which sleep state to enter

### 4.2.6 Escaping Infinite Loops: Interrupts
- The operating system generates timer interrupts periodically, and the CPU detects them and executes an interrupt handler within the OS
- The interrupt handler determines whether a process is ready to run, and if so, resumes the previously suspended process
- If not ready, it suspends the process and schedules another ready process
- halt + interrupt: even during the halt loop, timer interrupts occur periodically → the CPU is interrupted → if there is a ready process, it is executed → otherwise, the halt loop continues

## 4.3 How Does the CPU Understand Numbers?

### 4.3.1 The Number 0 and Positive Integers
- In Arabic numerals, the value depends on the position of digits, which is called positional notation
- Binary also uses positional notation
- With k bits, it is possible to represent 2^k unsigned integers

### 4.3.2 Signed Integers
- To represent signed integers, half of the representable values are used for negative numbers
- The most significant bit determines the sign
    - 0xxxxxx → positive
    - 1xxxxxx → negative

### 4.3.3 Sign-Magnitude Representation
- A method that keeps the magnitude unchanged while using the most significant bit to indicate the sign
- Example: 0001 → 1, 1001 → -1

### 4.3.4 One’s Complement
- A method that represents negative numbers by flipping all bits of the positive number
- Example: 0001 → 1, 1110 → -1

### 4.3.5 Adding Numbers Is Not So Simple
- Problem with sign-magnitude representation
    - 2 + (-2)
    - 0010
    - 1010 +
    - -----
    - 1100 → -4 (incorrect result)
- Using one’s complement
    - 2 + (-2)
        - 0010
        - 1101 +
        - -----
        - 1111 → -0 (not contradictory, but problematic)

### 4.3.6 A Computer-Friendly Representation: Two’s Complement
- We need a representation that satisfies A + (-A) = 0 and has only one representation for zero (0000)
    - 0010
    - 1??? +
    - -----
    - 0000 → real 0
- The correct value for 1??? is 1110, ignoring carry
- This allows adders to perform calculations without considering the sign
- Two’s complement
- A negative number in two’s complement is obtained by adding 1 to its one’s complement
    - Example: decimal -7
        - one’s complement → 1000
        - two’s complement → 1001
    - Example: decimal -4
        - one’s complement → 1011
        - two’s complement → 1100

### 4.3.7 Does the CPU Really Understand Numbers?
- This method is not intuitive for humans but simplifies circuit design
- The adder does not need to consider sign; it only needs results of logical operations such as AND and OR
- Only the compiler needs to understand whether values are in one’s complement or two’s complement
